## ü§ù How to Contribute

### For Researchers
- Share multilingual voice datasets (with proper consent)
- Validate models in your region/language
- Contribute clinical expertise
- Propose improvements to screening protocols

### For Developers
- Improve mobile app performance
- Add language support
- Optimize ML models for edge devices
- Fix bugs and improve code quality

### For Healthcare Workers
- Test the tool in real-world settings
- Provide feedback on usability
- Suggest culturally appropriate adaptations
- Report issues or limitations

### For Community Members
- Provide feedback as users
- Help with translations
- Share the project with communities that could benefit
- Participate in pilot studies (with proper ethical approval)

## üìã Contribution Guidelines

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
4. **Test thoroughly** (especially on low-end devices)
5. **Submit a pull request**

### Code Standards

- Python: Follow PEP 8
- Flutter/Dart: Follow official Dart style guide
- Comment complex algorithms
- Include unit tests for new features
- Update documentation
